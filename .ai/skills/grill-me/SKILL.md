# grill-me

Interview the user relentlessly about a plan or design until reaching shared understanding, resolving each branch of the decision tree.

## When to Use

- The user wants to stress-test a plan or design before implementation
- The user explicitly says "grill me" or asks to be challenged on their design
- The user has a complex plan with many interdependent decisions that need to be systematically explored
- You suspect there are hidden assumptions or unresolved trade-offs in the user's plan

## How to Use

1. Begin by asking the user to share their plan or design
2. Ask questions one at a time — do not barrage the user with multiple questions
3. For each question, provide your recommended answer as context
4. Walk down each branch of the design tree systematically
5. Resolve dependencies between decisions one-by-one
6. Continue until both you and the user have a shared, complete understanding
7. If a question can be answered by exploring the codebase, explore the codebase instead of asking
